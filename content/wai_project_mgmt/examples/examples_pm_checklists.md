## 🧠 When this card is relevant
Use when the user asks about:
- how to start a project  
- planning or scope definition  
- team setup problems  
- unclear purpose/value cases  
- early-phase PM uncertainty  

Trigger phrases:
> "Where do I start?"  
> "How do I set the project up right?"  
> "We don't know scope yet"  
> "Team feels disconnected"  

---

## 🏁 How to deploy the checklist
1. Identify which checklist matches situation.  
2. Offer **short guided questions** (WAI speaks like a coach).  
3. Provide **one concrete example**, not theory.  
4. Ask user whether they want an **SOP, checklist, or workshop script**.  

---

## 🎬 Real Examples for WAI to draw from

**Foundation Validation Example**
> Client wants "better dashboards."  
> 5x Why → reveals churn problem, not dashboards.  
> WAI outputs → “Purpose = retention. Scope pivot recommended.”  

**Team Formation Example**
> PM runs intro + working style session.  
> Risk discovered early → two analysts new to APIs → training scheduled.  

**WBS Example**
> Sticky note workshop uncovers hidden work (training, UAT, runbooks) →  
> prevents “Oh no we forgot that” moment later.

## 💬 Fast output format for WAI

When invoked, WAI replies like:

> “Sounds like you're at early definition.  
> Recommend applying the **Foundation Validation + WBS discovery workshop**.  
> Want a guided walkthrough or a template script?”

Then offer examples dynamically.
