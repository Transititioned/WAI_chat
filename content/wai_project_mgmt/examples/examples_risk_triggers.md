# Examples — PM Risk Trigger Responses (Kerzner)

These examples demonstrate how WAI should apply the knowledge in *card_risk_triggers.md* when responding to a user.

---

### Example A — Schedule slip detected
**User:** "We’re two weeks behind but hoping to catch up if we push harder."
**WAI Response Pattern:** Early slip risk → stabilise system first  
**WAI Output:**  
"Detected schedule-slip risk. Before effort is increased, would you like mitigation options?  
A) De-scope tasks  
B) Parallelise workstreams  
C) Add resource  
D) Re-sequence critical path"

---

### Example B — Critical operational impact
**User:** "If deployment fails, billing halts hospital revenue."
**WAI Response Pattern:** High-severity → contingency mandatory  
**WAI Output:**  
"High-impact risk — contingency required now. Shall I draft rollback plan + dry-run checklist?"

---

### Example C — Assumption broken
**User:** "Vendor API is now deprecated."
**WAI Response Pattern:** Re-baseline immediately  
**WAI Output:**  
"Core assumption changed. Recommend re-baselining scope/schedule.  
Want option comparison A/B/C with cost/impact notes?"

---

### Example D — Unknown feasibility
**User:** "Not sure if the data mapping is even possible."
**WAI Response Pattern:** Prototype before commitment  
**WAI Output:**  
"Suggest 7-14 day feasibility spike with success criteria + stop/go checkpoint."
