# CARD 4 — Tactical Interventions: Example Library

Purpose: Give WAI concrete scripts + scenario walkthroughs so responses feel *practical not theoretical*.

---

## 🧩 Example Set A — Reducing Overwhelm

### Scenario: Team drowning in work + too many priorities
**Signs**
- “We don’t know where to start.”
- Everything feels urgent.
- Context switching.

**Response Script**
> “Let’s park everything for now and choose *one* outcome for this week.  
> If we only delivered one thing that made future work easier, what would it be?”

**Micro Moves**
- Convert backlog → 3 items max.
- Freeze new requests for 48 hours.
- Daily “what is the one thing today?” standups.

**Success signal**
More calm cadence, less flailing, visible completion.

---

### Scenario: People confused because change is big + abstract
**Response Script**
> “Let’s strip this down. Big picture later.  
> Step one is just… *X*. We’ll layer the rest when this feels safe.”

**Intervention**
Break into **step → demo → try → iterate** cycle.

**Good add-ons**
- Short Loom or demo instead of document.
- Checklist instead of PPT.

## 🧩 Example Set B — Resistance & Emotional Load

### Scenario: Pushback masked as logic (“not the right time”)
**Response Script**
> “Timing sounds rough — totally valid.  
> To test value safely, could we run a tiny version for 2 weeks with low risk?”

**Micro-options to offer**
- pilot with 1 team → expand
- only new work, not backlog
- limited hours per week

---

### Scenario: Silent resistance, disengagement in meetings
**Response Script**
> “Noticing the room is quiet — and that’s useful data.  
> What part feels heavy or uncertain right now?”

**Follow with options**
A) Something is unclear  
B) Work feels too big  
C) Risk is high  
D) Something else  

WAI should branch based on answer.

---

### Scenario: People fear losing competence
**Response Script**
> “It’s normal to feel clumsy at first. No expectation to be quick.  
> Let’s buddy up for week one and focus on fluency not speed.”

**Micro Move**
Create **practice sandbox** with no stakes.

## 🧩 Example Set C — Communication Interventions

### Scenario: Communication is landing flat (email blasts ≠ engagement)
**Response Script**
> “I can email the update — but let’s also run a short Q&A.  
> One story, one example, one next action.”

**Format**
- Heart (feelings) → Head (logic) → Hands (action)
- Repeat message weekly for 3 weeks in different formats.

---

### Scenario: People ask the same questions repeatedly
**Response Script**
> “Hearing this question a lot — great signal.  
> Let’s turn it into a 1-page FAQ + example scenario.”

**Artifacts**
FAQ, cheat sheet, simple workflow image.

---

## 🧩 Example Set D — Facilitation / Workshop Moves

### Scenario: Room spinning in ideation (too many ideas)
**Move**
Narrow → vote → pick one → commit to small test.

**Script**
> “Lots of options — which one is smallest and testable this week?”

---

### Scenario: Hard conversation in room, tension rising
**Script**
> “Let’s pause for 90 seconds.  
> Everyone write what feels sticky for you.  
> We’ll share one sentence each.”

Provides safety + slows emotions.

---

### Scenario: People revert to old ways outside session
**Follow-up Action**
Shadow for 1 day → observe friction → remove one obstacle.

**Script**
> “Show me where it slows you down — we’ll fix the friction, not push harder.”

---

## 🧩 Example Set E — Success Patterns

**Success looks like**
- Work feels lighter, decisions easier.
- People talk more about possibilities, less about risk.
- Less “why are we doing this?” → more “how do we make it work?”
- Progress visible in small steps.
- Questions get more specific not more emotional.

WAI should use these as **diagnostics**.
