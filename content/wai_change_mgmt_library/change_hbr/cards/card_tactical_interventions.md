# CARD 4 — Tactical Interventions for Change
_Practical behaviours, scripts, moves that reduce friction in the field._

---

## 1) Core Logic of Tactical Interventions
When cognitive friction is high:
- reduce load → increase clarity → create safety → stabilise pace.
- people need **less info, more sense-making**.
- interventions must be **felt before understood**.
- progress is created through **small safe forward steps**.

Design rule:
> **Make the next step obvious, easy, and supported.**

---

## 2) Quick Intervention Patterns (Do This When… → Action)

| Situation | Intervention Move |
|---|---|
| People are overwhelmed | Reduce decisions to 1–3. Time-box. Park everything else. |
| Work feels “too big” | Break into **micro-wins**, visible incremental progress. |
| Confusion / ambiguity | Summarise shared reality → ask one clarifying question. |
| High resistance | Listen first, validate concerns → return later with options. |
| Lack of ownership | Ask: “What part of this could you own?” |
| Team emotionally overloaded | Pause, name the feeling → re-scaffold the decision. |
| Stalled conversation | Switch format: 2-person breakout → regroup. |
| Mistrust | Increase transparency → share “what we know / don’t know yet.” |
| Low engagement | Ask for input: “What would make this easier?” |
| Fear of incompetence | Offer learning safety: “No one is expected to be fast yet.” |

---

## 3) Leader Actions That Lower Friction

- Give **choice wherever possible** (even trivial choice rebuilds agency).
- Create **islands of stability** (what stays the same).
- Use **scaffolded adoption** → pilot → expand.
- Use **job aids, checklists, cheat sheets** to offset memory cost.
- Repeat key messages using **different channels**.
- Celebrate **small evidence of progress**, not perfection.
- Normalise the learning dip → “slow is expected before smooth.”
- Convert “initiative language” into **human words**.
- Make help **easy to access** (office hours, Slack channel, drop-ins).

---

## 4) Implementation Tactics — Fast Operational Moves

**Day-1 Actions**
- Publish change in **three sentences max**.
- Show *one example of success*, not the whole design.
- Ask team: “What feels unclear, heavy, unfair, risky?”

**First fortnight**
- Create a **one-page how-to guide**.
- Hold one **listening post** and extract top 3 friction points.
- Convert friction → backlog → visible burn-down.

**Sustainability**
- Turn recurring questions into **micro-FAQs**.
- Track **emotion patterns over time** (frustration → curiosity → pull).

---

## 5) Simple Phrases WAI Can Recommend

- “Let’s shrink this down — what’s the next *smallest* step?”
- “You don't need the whole picture today — just step one.”
- “Sounds like load is high. One thing at a time.”
- “What would make this change feel safe enough to try?”
- “What part can you own that feels doable?”
- “Tell me where the friction is — workflow, clarity, emotion, skill?”

---

## 6) Scenarios & Micro-Playbooks

### Scenario: Resistance masked as “bad timing”
**Move:** Validate → narrow → offer experiment.  
> “Makes sense the timing feels rough.  
> Rather than all at once — what if we trial a light version for 2 weeks?”

### Scenario: People reverting to old ways
**Move:** Reinforce safety + remove friction.  
> “Old workflow is familiar — learning new one costs energy.  
> What step is slow or unclear? Fix that first — not the whole rollout.”

### Scenario: Team anxious about competence
**Move:** Normalize learning dip.  
> “Feeling clumsy is normal — it means you’re learning.  
> No expectation to be fast yet. Let’s buddy-pair for week one.”

### Scenario: Communication chaos
**Move:** Collapse to one source of truth.  
> “We’ll post updates only in Teams channel X — no other threads.”

## 7) How WAI Should Use This Card

When user asks:
> “How do I reduce resistance / unblock a stuck team / make adoption smoother?”

WAI should respond with:
1. **1–3 recommended interventions**, not a brain dump.
2. A clarifying question first if intent unclear.
3. Optionally: Script example + micro checklist + decision branch.

Example output pattern:
> “Three moves worth trying are…  
> If you want, I can also draft the exact message/script to send.”
